## Sub-task 2: `NewsItem` model — `src/models/news.py`

**Fields (from design-doc Section 4.1):**
| Field | Type | Default |
|-------|------|---------|
| `id` | `str` | `uuid4()` factory |
| `source` | `str` | required |
| `title` | `str` | required |
| `content` | `str` | required |
| `url` | `str \| None` | `None` |
| `published_at` | `datetime` | required |
| `tickers` | `list[str]` | required |
| `fetched_at` | `datetime` | `datetime.utcnow()` factory |

**Patterns:**
- Use `Field(default_factory=...)` for `id` and `fetched_at`
- UUID generation: `lambda: str(uuid.uuid4())`

## Sub-task 8: Unit tests — `tests/test_models.py`

**Test cases per model:**

### NewsItem
- Create with all required fields — succeeds, `id` and `fetched_at` auto-populated
- Create with `url=None` — succeeds (optional field)
- Verify `tickers` accepts `["BTC", "ETH"]`
- `model_dump()` returns dict with all fields
- `model_dump_json()` produces valid JSON string, `model_validate_json()` round-trips back

### AnalysisResult
- Create with `sentiment="bullish"` — succeeds
- Create with `sentiment="invalid"` — raises `ValidationError`
- Verify `analyzed_at` auto-populates

### ContentVariant
- Create with `score=None` — succeeds (unpopulated before scoring)
- Create with `style="analytical"` — succeeds
- Create with `style="unknown"` — raises `ValidationError`
- Verify `score_breakdown` accepts `{"hook_strength": 8, "clarity": 7, "engagement": 6, "relevance": 9}`

### DistributionRecord
- Create with only `variant_id` — succeeds, defaults: `status="pending"`, `platform="twitter"`
- Create with `status="posted"` — succeeds
- Create with `status="invalid"` — raises `ValidationError`

### PipelineRun
- Create with `trigger="cli"` — succeeds, defaults: `status="running"`, counters at 0
- Create with `trigger="webhook"` — raises `ValidationError`
- Verify `started_at` auto-populates
- Test updating `status` to `"completed"` with `model_copy(update={...})`

### Shared patterns
- Use `pytest.raises(ValidationError)` for negative cases
- Each model class gets its own test class (e.g., `TestNewsItem`)
- Helper fixture `sample_news_item()` etc. for reuse across tests
