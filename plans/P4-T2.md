# P4-T2: Enable Real Posting with DistributionRecord Tracking

## Overview

Wire `post_tweet()` into the API endpoints, pipeline, and CLI. Track `DistributionRecord`s in-memory and update `PipelineRun.variants_posted`.

## Sub-tasks

### 1. Wire `/post` endpoint to `post_tweet()` — small
- **Why:** Currently returns a stub "pending" record without calling Twitter.
- **What:** In `src/api/app.py`, import `post_tweet`, call it with the found variant, store the `DistributionRecord` in a new `_distributions` dict.
- **Test:** Unit test mocking `post_tweet` to verify the endpoint returns a real record.

### 2. Wire `/post/batch` endpoint to `post_tweet()` — small
- **Why:** Same stub issue for batch posting.
- **What:** Loop over variant IDs, call `post_tweet()` for each found variant, collect records.
- **Test:** Unit test with mock.

### 3. Add distribution tracking store — small
- **Why:** Need to persist records in-memory so they can be queried.
- **What:** Add `_distributions: dict[str, DistributionRecord] = {}` to `app.py`. Store each record after posting. Records are returned inline from `/post` and `/post/batch`.
- **Test:** Covered by endpoint tests.

### 4. Integrate distribution into `run_pipeline()` — medium
- **Why:** Design doc says pipeline should distribute top picks. Currently stops at scoring.
- **What:** In `src/pipeline.py`, after selecting top variants, call `post_tweet()` for each. Update `run.variants_posted`. Return `DistributionRecord`s alongside existing return values (update signature to `tuple[PipelineRun, list[ContentVariant], list[DistributionRecord]]`).
- **Reuse:** `post_tweet()` from `src/modules/distribution.py`, graceful degradation pattern (errors don't halt pipeline).
- **Test:** Mock `post_tweet` in pipeline tests.

### 5. Update CLI display — small
- **Why:** CLI should show posting outcomes.
- **What:** In `src/cli.py`, after printing variants, show distribution status for each.
- **Test:** Existing CLI test pattern.

### 6. Update tests — medium
- **Why:** Verify all wiring works.
- **What:** Update `tests/test_distribution.py`, `tests/test_api.py`, `tests/test_pipeline.py`, `tests/test_cli.py`.
