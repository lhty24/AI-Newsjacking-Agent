# P2-T3: Add Error Handling (Retries, Graceful Degradation)

## Overview

Enhance the existing error handling across the pipeline with retry visibility, structured API error responses, config validation, rate-limit awareness, and partial-failure tracking.

## Sub-tasks

### 1. Enhance tenacity retries with logging callbacks

**Why:** Retries happen silently â€” no visibility into transient failures.

**What:** Add `before_sleep` logging to all `@retry` decorators so each retry attempt is logged with the exception and attempt number.

**Files:** `src/modules/ingestion.py`, `analysis.py`, `generation.py`, `scoring.py`

**Complexity:** Small

### 2. Add FastAPI error handling middleware

**Why:** Unhandled exceptions return raw 500s. Need structured error responses and logging.

**What:** Add a global exception handler and request validation error handler to `app.py`. Return consistent JSON error shapes.

**Files:** `src/api/app.py`

**Complexity:** Small

### 3. Add config validation

**Why:** Missing `LLM_API_KEY` causes cryptic failures deep in the pipeline. Should fail fast.

**What:** Add a `validate_config()` function in `src/config.py` that checks required env vars. Call it at API startup (FastAPI lifespan) and in CLI.

**Files:** `src/config.py`, `src/api/app.py`, `src/cli.py`

**Complexity:** Small

### 4. Add rate-limit-aware retry predicate

**Why:** 429 errors need different backoff than transient 500s. CoinGecko and LLM APIs return 429 when rate-limited.

**What:** Add a custom `retry_if` predicate that uses longer backoff for rate-limit errors (429/RateLimitError). Apply to ingestion and LLM retry decorators.

**Files:** `src/modules/ingestion.py`, `analysis.py`, `generation.py`, `scoring.py`

**Complexity:** Small

### 5. Improve pipeline partial-failure tracking

**Why:** Pipeline currently logs counts but doesn't track which items failed or expose failure info in the `PipelineRun` result.

**What:** Add an `errors` field to `PipelineRun` model to capture per-stage failure counts. Update pipeline orchestrator to populate it.

**Files:** `src/models/pipeline.py`, `src/pipeline.py`

**Complexity:** Small

### 6. Add tests for error handling paths

**Why:** Verify retries, graceful degradation, and fallbacks work as expected.

**What:** Tests for: retry exhaustion returns gracefully, partial batch failures, config validation, API error responses.

**Files:** `tests/test_error_handling.py`

**Complexity:** Medium

## Testing

- Unit tests for config validation (missing key, empty key)
- Unit tests for retry exhaustion with mocked external calls
- Unit tests for partial batch failures (some items fail, others succeed)
- Integration test for API error handler returning structured JSON
- Verify pipeline `errors` field populated correctly on partial failures
