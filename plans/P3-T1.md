# P3-T1: Build Streamlit Dashboard

## Overview

Build an interactive Streamlit dashboard that displays news, pipeline runs, content variants with scores, and allows triggering the pipeline and posting variants. Calls existing FastAPI endpoints.

## Sub-tasks

### 1. Add Streamlit dependency

**Why:** Streamlit is not in requirements.txt yet.

**What:** Add `streamlit>=1.30` to `requirements.txt`.

**Test:** `pip install -r requirements.txt` succeeds.

**Complexity:** Small

### 2. Build the dashboard app

**Why:** Core deliverable — interactive UI for the newsjacking agent.

**What:** Create `src/dashboard.py` using Streamlit with:
- Sidebar with navigation and pipeline trigger button
- **Dashboard** page: latest run summary, key metrics
- **Runs** page: table of pipeline runs from `GET /runs`, click to expand details
- **News** page: display from `GET /news` with ticker badges
- **Variants** page: cards showing variant text, style, score breakdown from `GET /variants`
- **Post** page: select variants and post via `POST /post`

**Patterns to reuse:**
- Data models from `src/models/` for type reference
- API endpoints from `src/api/app.py`

**Test:** Run `streamlit run src/dashboard.py` manually, verify all pages render.

**Complexity:** Large

### 3. Add API base URL config

**Why:** Dashboard needs to know where the FastAPI server is running.

**What:** Add `API_BASE_URL` to `src/config.py` (default `http://localhost:8000`).

**Test:** Config loads correctly with default and custom values.

**Complexity:** Small

### 4. Add dashboard tests

**Why:** Verify the API client functions fetch and parse data correctly.

**What:** Test helper functions that call API endpoints with mocked httpx responses.

**Test:** `pytest tests/test_dashboard.py`

**Complexity:** Medium

## Data Models to Display

- **NewsItem**: id, source, title, content, url, published_at, tickers, fetched_at
- **PipelineRun**: id, trigger, started_at, completed_at, status, news_count, variants_generated, variants_posted, error, stage_errors
- **ContentVariant**: id, analysis_id, style, text, prompt_template, score, score_breakdown, generated_at
- **DistributionRecord**: id, variant_id, platform, status, posted_at, error

## API Endpoints to Consume

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/news` | GET | Fetch latest crypto news |
| `/run` | POST | Trigger full pipeline |
| `/runs` | GET | List recent pipeline runs |
| `/runs/{run_id}` | GET | Get specific run + variants |
| `/variants` | GET | List variants (optional run_id filter) |
| `/post` | POST | Post a specific variant |
| `/post/batch` | POST | Post multiple variants |
