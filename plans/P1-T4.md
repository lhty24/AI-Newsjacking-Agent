# P1-T4: Implement Content Generation with Multi-Style Templates

## Context

The pipeline has news ingestion (P1-T2) and LLM-based analysis (P1-T3). This task adds the content generation stage: given an `AnalysisResult`, generate multiple tweet variants in different styles (analytical, meme, contrarian) using style-specific prompts and temperature settings.

## Implementation Steps

### Step 1: Create `src/modules/generation.py`

#### Style Configuration

- `STYLE_PROMPTS` dict — maps each style to a system prompt instructing the LLM to write a tweet
- `STYLE_TEMPERATURES` dict — `{"analytical": 0.3, "meme": 0.9, "contrarian": 0.7}`

#### Style Prompts

Each prompt ends with: "Return JSON with a single field `text` containing the tweet (max 280 chars)."

- **analytical**: "You are a crypto market analyst writing a professional tweet. Be data-driven, clear, and authoritative."
- **meme**: "You are a crypto Twitter personality. Write a viral, funny tweet using crypto slang, emojis, and meme culture."
- **contrarian**: "You are a contrarian crypto analyst. Challenge the mainstream narrative. Be provocative but substantive."

#### Internal Functions

- `_build_generation_prompt(analysis: AnalysisResult) -> str` — formats analysis fields (sentiment, summary, topics, signal) into the user prompt
- `_call_llm_with_temperature(system_prompt, user_prompt, temperature) -> str` — LiteLLM completion with retry, accepts temperature override (analysis module hardcodes temperature from config; generation needs per-style control)
- `_generate_single(analysis: AnalysisResult, style: str) -> ContentVariant` — generates one variant, parses tweet text from JSON response, sets `prompt_template` to the style name

#### Public Function

- `generate_variants(analysis: AnalysisResult, styles: list[str] | None = None) -> list[ContentVariant]` — defaults to all 3 styles, loops with graceful degradation (log warning on failure, continue), returns list of successful variants

#### Patterns to Reuse

- Retry decorator: `@retry(stop=stop_after_attempt(3), wait=wait_exponential(min=1, max=10))` from `src/modules/analysis.py`
- Config imports: `LLM_MODEL`, `LLM_API_KEY`, `LLM_MAX_TOKENS` from `src/config.py`
- JSON parsing with markdown fence stripping from `src/modules/analysis.py`
- Output model: `ContentVariant` from `src/models/content.py` (fields: `analysis_id`, `style`, `text`, `prompt_template`)

### Step 2: Update `src/modules/__init__.py`

Add `generate_variants` to imports and `__all__`.

### Step 3: Create `tests/test_generation.py`

Mock at `_call_llm_with_temperature` level. Follow patterns from `tests/test_analysis.py`.

#### Test Cases

1. Single style generates correct `ContentVariant` with matching style and analysis_id
2. Default styles (None) generates all 3 variants
3. Partial failure — one style fails, other 2 succeed
4. All fail → empty list
5. Prompt includes analysis fields (sentiment, summary, signal, topics)
6. Style temperature mapping is correct (verify temperature passed to LLM call)
7. Generated text extracted from JSON response correctly
8. Markdown code fences in response are handled

### Step 4: Run tests and verify

pytest tests/test_generation.py -v
pytest tests/ -v  # no regressions
