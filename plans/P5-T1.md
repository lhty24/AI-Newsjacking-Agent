# P5-T1: Add Scheduler for Periodic Execution

## Context

Phases 1–4 are complete. The next task is P5-T1: adding APScheduler so the pipeline runs automatically at a user-configurable interval. The scheduler calls the same `run_pipeline()` function used by CLI and API, storing results in the same in-memory stores so scheduled runs appear in the dashboard.

**User requirement:** Interval selector as a dropdown (1, 3, 8, 12, 24 hours, default 12h) above a start/stop button in the dashboard sidebar.

## Implementation Steps

### 1. `requirements.txt` — Add dependency
Add `apscheduler>=3.10` (v3.x, not v4 which is still alpha).

### 2. `src/config.py` — Add config vars
```python
SCHEDULER_ENABLED = os.environ.get("SCHEDULER_ENABLED", "false").lower() == "true"
SCHEDULER_INTERVAL_HOURS = int(os.environ.get("SCHEDULER_INTERVAL_HOURS", "12"))
```

### 3. `src/scheduler.py` — Create scheduler module

Uses `BackgroundScheduler` with `IntervalTrigger`. Exposes:

- `init_scheduler(callback, interval_hours)` — create scheduler + add job (paused)
- `start_scheduler()` / `stop_scheduler()` — control execution
- `update_interval(interval_hours)` — reschedule the job
- `get_scheduler_status() -> dict` — returns `{running, interval_hours, next_run_time}`

Key settings: `max_instances=1`, `coalesce=True`, `misfire_grace_time=3600`.

The callback is injected (not imported) to avoid circular imports with `app.py`.

### 4. `src/api/app.py` — Endpoints + lifespan

**Callback wrapper** (defined in app.py):
```python
def _scheduler_pipeline_callback():
    status = get_scheduler_status()
    run = PipelineRun(trigger="scheduler")
    _runs[run.id] = run
    _variants[run.id] = []
    _execute_pipeline(run.id, max_articles=status["max_articles"], trigger="scheduler")
```

**Endpoints:**
- `GET /scheduler/status` → `{running, interval_hours, max_articles, next_run_time}`
- `POST /scheduler/start` → start scheduler, return status
- `POST /scheduler/stop` → stop scheduler, return status
- `POST /scheduler/interval` → body `{interval_hours: int}`, validate against `[1,3,8,12,24]`, return status
- `POST /scheduler/max-articles` → body `{max_articles: int}`, validate against `[1,3,5,10]`, return status

**Lifespan integration:**
- On startup: `init_scheduler(callback, SCHEDULER_INTERVAL_HOURS)`, auto-start if `SCHEDULER_ENABLED`
- On shutdown: `stop_scheduler()`

### 5. `src/dashboard.py` — Sidebar scheduler controls

Add below existing "Run Pipeline" button:
```
st.divider()
st.subheader("Scheduler")
# Fetch status from GET /scheduler/status
# Articles dropdown: st.selectbox("Articles to process", [1,3,5,10], default 3)
```

Follows existing patterns: `api_get`/`api_post` helpers, `st.selectbox`, `st.button(use_container_width=True)`, `st.rerun()`.
